Gestion des Sources
et Lieux de stockage


Les informations qui sont à la base de toute généalogie peuvent être le résultat de recherches personnelles (Archives, notaires,...), de partage avec d'autres généalogistes, le recueil d'informations orales, papiers de famille, etc...
Quelle quâ€™en soit l'origine, tout bon généalogiste se doit d'afficher ses sources. Il n'est pas une généalogie sans sources.


Deux mots-clés
  • Le lieu de stockage (pour la norme Gedcom, REPO, de l'anglais "repository", lieu de stockage, de dépôt): C'est le lieu de dépôt physique d'un document original (mairie, archives départementales, etc... ou simplement un classeur chez vous).
  • La source (SOUR): c'est le document qui contient l'information (un registre paroissial, une table décennale, etc..).

Bien qu'il s'agisse de deux entités différentes, il est plus logique et plus efficient de créer le lieu de stockage avant de créer la source. D'une part la source est contenue dans le lieu de stockage, d'autre part, lors de la création d'une source vous devrez faire un lien vers son lieu de stockage.

Création d'un lieu de stockage

  • clic droit dans un endroit vide de la Table des entités, de l'Arbre dynamique ou d'un éditeur
  • sélectionner la ligne "Gedcom votre_fichier.ged", puis dans le sous-menu "Créer un (des) lieu(x) de stockage".
  • cliquer sur "Exécuter et créer un lieu de stockage".
  • l'éditeur Gedcom affiche une arborescence de ce type:


Cette fenêtre affiche un certain nombre d'étiquettes par défaut que vous pouvez modifier, pour l'adapter à vos éléments d'information. Vous allez donc gérer les étiquettes.

Gérer les étiquettes
  • Suppression d'une étiquette:
    • clic droit sur la ligne, puis dans le menu cliquer sur "Couper" (par exemple, pour supprimer STAE, inutilisé en France).
  • Ajout d'une étiquette:
    • un clic droit sur la 1ère ligne (1) "@Rx@ REPO", puis clic gauche sur "Liste des propriétés Gedcom" vous permet de choisir parmi des labels pré-établis (l'option tout en bas "Ajouter les sous-propriétés par défaut" doit être cochée). Cliquer sur "OK" pour confirmer.
      La "Liste des propriétés Gedcom" propose les étiquettes pouvant être utilisées dans le respect de la grammaire Gedcom, selon la cible pointée.
    • Un clic droit sur certaines étiquettes, puis clic gauche sur "Ajouter une propriété", vous proposera des sous-étiquettes parmi les étiquettes déjà utilisées à ce niveau.

Un lieu de dépôt est maintenant créé, il pourra être réutilisé pour l'ensemble de votre généalogie. Désormais, il apparaîtra dans le menu déroulant qui s'ouvrira lors des prochaines créations de "Lieux de stockage".
Les lieux de stockage sont listés dans l'Explorateur Gedcom et dans la Table des entités, d'où ils peuvent extraits pour être modifiés (orthographe, par exemple).

Création d'une source

  • clic droit dans un endroit vide de la Table, de l'arbre ou d'un éditeur
  • sélectionner la ligne "Gedcom votre fichier.ged'", puis dans le sous-menu, "Créer une (des) Source(s)"
  • cliquer sur "Exécuter et créer une Source".
  • l'éditeur Gedcom affiche une arborescence à compléter avec vos éléments d'information.



Petite explication de texte (2):

    • AUTH ( Auteur): comprendre "Dépositaire ou Créateur du document".
    • TITL (Titre): utilisé, ici, comme "Intitulé du document".

Vous pouvez modifier les étiquettes sur le même principe que pour REPO.

Il reste à localiser la source, c'est-à-dire créer un lien avec un lieu de stockage.
    • clic droit sur la 1ère ligne "@Sx@ SOUR" (1), descendre sur Source 'Sx'
    • dans le sous-menu, sélectionner "Créer ou établir une liaison vers un Lieu de Stockage".
    • dans le menu déroulant, repérer le lieu, confirmer par "Exécuter et Créer un Lieu de Stockage"
      • Une nouvelle ligne est créée dans la fenêtre de l'éditeur Gedcom (3).

Une source est maintenant créée, elle pourra être réutilisée pour l'ensemble de votre généalogie. Désormais, elle apparaîtra dans le menu déroulant qui s'ouvrira lors des prochaines créations de "Source(s)". Les différentes utilisations d'une source sont récapitulées en fin d'entité (4).
Les Sources sont listées dans l'Explorateur Gedcom et dans la Table des entités, d'où elles peuvent être affichées pour être modifiées (orthographe, par exemple).


Une bonne gestion des sources nécessite quelques précisions supplémentaires.

 Tout d'abord, il faut enregistrer sa côte(*) (CALN):

    • à partir de la fenêtre "SOUR": clic droit sur la ligne REPO, sélectionner "Ajouter une propriété".
    • un clic sur "Numéro de répertoire CALN" crée une zone de saisie.
 Puis, le type de support média utilisé pour cette source:
    • clic gauche sur la ligne "CALN" puis, sur "Liste des propriétés Gedcom" vous propose le label "MEDI".
    • un clic sur "Ok" crée une zone de saisie associée à un menu déroulant pré-établi (si besoin, vous pouvez ajouter des types de support).


(*)numéro qui sert à identifier une pièce dans une collection. Les normes de codifications nationales des A.D. ne sont pas toujours respectées. La cote peut être de ce type de format:


 Et pour finir d'enregistrer vos informations, créer une étiquette "données" (DATA en Gedcom). Vous savez faire, toujours le même principe:

    • clic droit sur la 1ère ligne "Liste des propriétés Gedcom", sélectionner "Données". Si l'option tout en bas "Ajouter les sous-propriétés par défaut" est cochée, vous aurez en prime sa sous-étiquette EVEN.
    • un menu déroulant est associé à EVEN, même pas besoin d'écrire. Choisir, confirmer par "OK".
      Vous pouvez utiliser cette zone à votre gré (par exemple, pour un registre contenant plusieurs types d'actes).
    • à partir de la ligne EVEN, clic droit pour "Ajouter une propriété Gedcom" et choisir "DATE" et "PLAC". Toujours "OK" pour confirmer.
    • Il ne vous reste plus qu'à compléter les zones de saisie.
      • On peut ajouter autant de sous-étiquettes que nécessaire.


Pour sélectionner deux étiquettes à la fois:
Ctrl (maintenu) + clic gauche sur chaque ligne.

Votre éditeur doit maintenant ressembler à ça: 

    @S1@ SOUR
      AUTH Archives Départementales du Cantal
      TITL Site Internet Arch. Dép. Cantal - Allanche - 1850-1859 - Naissances
      DATA
        EVEN BIRT
          DATE 1850-1859
          PLAC ,Allanche,15160,Cantal,Auvergne, France
      REPO Archives Départementales du Cantal (R1)
        CALN AD2E047/10
          MEDI electronic
      CHAN 2 Fév 2009, 13:06:24
L'ordre des étiquettes peut être différent.


Maintenant tout est prêt pour pouvoir passer à l'étape suivante Importer ou créer une généalogie.

II - Interface

  1. Q - Je suis assez dérouté par GenJ. Il n'a pas l'air de fonctionner comme les autres logiciels. Quelle approche avoir ?
    R - Le premier conseil que l'on peut donner, est qu'il ne faut pas essayer de l'utiliser comme les autres logiciels. GenJ est basé sur un multifenêtrage, donc il faut mieux utiliser plusieurs fenêtres en même temps (ex : Ouvrir à la fois les fenêtres de l'arbre généalogique, de la table et de l'éditeur). Ensuite il n'y a pas de barre de Menus avec des sous-menus déroulants ; au contraire, GenJ, utilise beaucoup les sous-menus mais au travers du clic droit de votre souris. Essayez, et ceci pratiquement partout, un clic droit vous ouvrira de multiples possibilités.
    Autre élément à prendre en considération  : GenJ est un trés puissant éditeur Gedcom, donc il est vraiment basé sur la structure interne du format Gedcom. Vous devrez vous habituer au langage utilisé, même si c'est vrai, des efforts importants sont faits pour franciser au maximum le logiciel. Les "tags" que l'on pourraient traduire par "étiquettes", voir "propriétés", et les "pointeurs", doivent être des notions qui tout doucement vous deviendront évidentes. Bien sûr, l'éditeur standard sera toujours là pour vous faciliter la tâche, mais l'éditeur gedcom restera toujours, à mon avis, le meilleur outil pour travailler de manière professionnelle sur votre arbre généalogique. N'hésitez pas à passer de l'un à l'autre ; vous apprendrez énormément, et vous construirez votre arbre exactement selon vos désirs.
    Enfin, GenJ demande un peu de temps pour être appréhendé. Ne soyez pas pressé, et donnez vous un peu de temps pour le découvrir, vous ne serez pas déçu.

  2. Q - Comment puis-je faire pour que GenJ utilise un autre langage ?
    R - Téléchargez le pack langage approprié (genj_ll-x.y.zip) pour GenJ (ex : genj_fr-2.3.3.zip pour le fichier français) depuis la Page Principale du Projet et suivez les instructions décrivant son installation. GenJ essaiera d'utiliser le langage de votre système automatiquement. Pour changer intentionnellement de langage (le pack langage doit être installé manuellement) choisissez le menu Visualisations, puis le sous-menu Options. Ensuite sélectionnez le langage de votre choix.
    Vous devrez redémarrer GenJ, pour que cela soit pris en considération.

  3. Q - Je voudrais que GenJ ait un autre look. Comment faire ?
    R - GenJ utilises la boite d'Outils Swing pour présenter les informations à l'utilisateur. Par défaut c'est la configuration de votre système qui sera prise en considération. D'autre part, GenJ vous permet également de changer le look, l'Apparence courante de l'interface.
    Pour cela, téléchargez le pack Apparence (genj_lnf-x.y.zip) depuis la page des projets de GenJ. Supposons que le fichier que vous avez récupéré s'appelle genj_lnf-2.0.zip, vous allez placer le fichier dans la directory d'installation de GenJ (sous un système Unix, ça pourrait être par exemple /home/pierre/GenJ/). Cette directory est celle o? vous avez installé le programme GenJ lui-même. Une fois le fichier dans cette directory, vous le décompactez, et vous allez voir qu'il va se créer une sous directory qui s'appellera "lnf".
    Après avoir redémarré GenJ, choisissez le menu Visualisations et allez dans le sous-menu Options, choisissez la langue que vous voulez, ex : Français. Maintenant, vous pouvez sélectionner un Aspect (et éventuellement un Thème), ex : Skin(arvernesthemepack.zip).

  4. Q - Comment puis-je faire pour qu'un centimètre ait cette taille sur mon écran ?
    R - GenJ utilise le sytème métrique mondial pour de nombreux composants (par ex. dans la Visualisation de l'Arbre Généalogique). Pour que les longueurs de lignes et les tailles des boîtes apparaissent correctement sur votre écran, vous aurez à indiquer à GenJ votre résolution d'écran (qui est donnée par le système mais normalement ne vous occupez pas des paramètres système du moniteur). Pour ajuster explicitement la résolution choisissez le sous menu du menu Aide|A propos et allez à l'onglet Apparence. Ensuite utilisez une règle correspondante à celles montrées pour votre écran.

  5. Q -  Pourquoi les images sont elles affichées différemment ?
    R - Les empreintes ou calques jouent un rôle important dans le rendu des images. En fonction de ce qui doit être affiché dans un cadre, l'espace alloué à une image peut être limité, et donc cette dernière peut être réduite pour laisser de la place au texte. Si il y a assez de place, l'image prendra toute la place disponible jusqu'à concurrence de sa taille réelle (déterminée en fonction de sa taille et de sa résolution).
    Voilà pourquoi plusieurs images peuvent avoir une allure différente.

  6. Q -  Comment fonctionnent les signets (bookmarks) ? Il existe bien une fonction ajouter/établir un signet, mais comment/où les utiliser ensuite ?
    R - Les signets peuvent être créés par un clic droit sur une personne, une famille.
    Dans le menu contextuel qui s'ouvre, il existe une option "Ajouter un signet". Si vous confirmez, il vous est présenté une fenêtre de saisie où vous pouvez mettre le nom que vous voulez à ce signet. Un nom par défaut est proposé, mais il ne correspond peut être pas à ce que vous voulez, d'où l'intérêt d'y mettre un nom qui va vous dire quelque chose par la suite, qui sera plus parlant.

    Une fois que c'est fait, si vous regardez la fenêtre de visualisation de l'arbre généalogique, vous avez une petite icone en bas de l'écran qui ressemble à un petit bonhomme avec un drapeau. Cliquez dessus, vous allez voir la liste de vos signets. Cliquez sur l'un d'eux, automatiquement l'arbre va faire que cette personne/famille sera la racine de votre arbre. Cette fonction est pratique car elle permet de travailler trés facilement sur l'arbre. Mettez des signets sur toutes les personnes/familles que vous voulez, et d'un clic, votre arbre se centre sur elle.
    Ensuite vous pouvez en supprimer, ou modifier l'ordre de la liste de vos signets. Pour cela il suffit d'aller dans la fenêtre d'Edition des paramètres de l'arbre généalogique (une icone à droite de celle que j'indiquais plus haut - elle me fait penser à une boite de sardines, mais en fait il s'agit d'une scie ;-) et d'un tournevis, quoique je n'en sois pas sûr). Vous cliquez dessus, et vous allez dans le dernier onglet. Là vous verrez tous vos signets, et les options sont suffisamment explicites pour voir ce qu'on peut en faire.

  7. Q -  Je voudrais utiliser GenJ sur une simple clé USB par exemple, mais j'ai remarqué qu'à chaque changement de machine, je laissais tout un tas de petits fichiers. J'aimerais bien que GenJ soit plus discret et qu'il mette tout ça sur la clé elle-même. Est-ce possible ?
    R - Oui, c'est trés facile. Il suffit de modifier un peu le script de lancement.
    Sous Windows, par exemple, vous pouvez mettre :

    javaw -Duser.home=. -Xmx512m -Xms32m -jar run.jar
    Le paramètre important, pour cette question est : "-Duser.home". On voit qu'ici on met juste un ".", ce qui veut dire que les fichiers seront créés dans un sous répertoire .genj situé en dessous du répertoire courant, donc tout restera sur la clé.
    Sous Linux, il faut faire :
    java -Duser.home=. -Xmx512m -Xms32m -jar run.jar
    Sympa, non!

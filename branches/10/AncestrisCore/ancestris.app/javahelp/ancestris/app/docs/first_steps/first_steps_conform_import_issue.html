Identified importing issues


Many softwares don't respect the Gedcom standard, it is very likely for you to find a few problems when migrating from other software. We can not talk about what we don't know, only about what already happened. Help us improve Ancestris, reporting issues you find at the discussion list.

Issues with Heredis

  • Heredis exports empty tags: it's necessary to add a Y, at least for the main ones (DEAT, MARR, MARC, DIV).
  • Two big issues with dates:
    • French republican years are exported as roman numbers (example, "15 NIVO an XII", while Gedcom standard demands "15 NIVO 12");
    • bad management of the calendar indicator (Julian, Gregorian, Republican, ...): the exported dates to French Republican calendar does not respect the standard. The indicator @DFRENCH R@ should be before the date, but BEFORE the positioning indicator (BEF for example, to say "before this date", etc). Heredis always puts the calendar indicator before the DATE tag in the Gedcom file, even if we have ABT, BET...AND. So, we should have: 2 DATE BEF @DFRENCH R@ 15 NIVO 12 (Gedcom format) and not 2 DATE @DFRENCH R@ BEF 15 NIVO an XII (Heredis format).
      The Julian calendar indicator, that should always be exported before 1582, can be suppressed in this case.
  • Tags out of context:
    • DATE under OJBE.
    • TIME under DATE everywhere.
    • ADDR under INDI/FAM without RESI.
  • Custom Tags (tags not belonging to Gedcom standard but allowing a genealogist to add additional information) without "_" (underscore): WWW (web site), EMAIL (email address), HEAL, etc.
  • Heredis custom fields: it's exported 1 TAG value with the TAG to be chosen from a list of standard Gedcom out of context. A correspondence must be foreseen.
  • Sources: bad management of sources and repositories.
Ancestris solution: install the plug-in "Imports – Heredis Import".

Issues with Généatique

Currently we have no feedback on importing files from Généatique. This does not mean there aren't some issues, just that no user of Généatique reported them.

Ancestris solution: report your issue in the discussion list.

Issues with Genealogie.com

  • The files created by the genealogy site genealogie.com don't have the information relative to the SUBMitter (the person who collected the informations present in the genealogy file). Such indication is mandatory for compliance with the standard.
Ancestris solution: Create the information going to Tools/Preferences/Data/Default supplier

Issues with LifeLines

  • With the default settings, LifeLines does not export the SUBM tag, which is mandatory.
Ancestris solution: Create the information going to Tools/Preferences/Data/Default supplier

Issues with Family Tree Builder

  • Family Tree Builder (FTB) generates numerous own Tags, that render the Gedcom file less readable;
Ancestris solution: install the plug-in "GEDCOM - remove tag".

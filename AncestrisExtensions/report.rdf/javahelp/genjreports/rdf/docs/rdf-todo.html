Developing netbeans GUI's is beyond my scope so I hope somebody else jumps in. New features are available as a library with command line functions. See http://gedcom2sem.googlecode.com/

current plugin status: proof of concept

This plugin is delivered as a proof of concept. Though it is even more than a report framework on its own, part of the functionality is squeezed into the user interface of an existing report framework.

Known bug in the report view: You might have to run a context-free query before you can run a report that should be launched from an entity.

The future

The main classes of the new library are a good starting point to plug the new export functions back into Ancestris. Adding RDFa to generated web pages would be a next step. Note that a mistake in a query might cause it to execute forever. So it should be possible to abort exports.

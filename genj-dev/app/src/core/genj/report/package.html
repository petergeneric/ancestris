Reports are simple, plug-able Java Types which are executed in the context of GenealogyJ. They can access the underlying object-representation of Gedcom-files opened by the user.

A selection of reports comes with the GenealogyJ distribution and can be found in the report-directory inside the GenJ installation directory.

Running Reports

There are multiple ways of intergrating a report with GenJ. Generally every report found in the report-directory (or one of its sub-directories) will be loaded and displayed in ReportView. There the user can read about the report's author and operation description, set report-options and run them individually.

A report can also be run by opening the context-menu for a Gedcom element in the UI. Depending on the selected element (for example an entity, a property or a Gedcom-file itself) reports that can accept this context as input are presented. A report declares whether it can run on a given context by overriding 

 String accepts(Object context)
and returning a textual description of the action the report will do if executed (if applicable - null otherwise).

By default it's assumed that a report can be executed on instances of the Gedcom type - when run the report is called with a reference to the instance of the corresponding type.

Report In/Output

Reports have several ways of interacting with the user - they can generate output:

  • print text to a stream that is displayed in ReportView's report output widget (see println())
  • open a browser on a URL (see showBrowserToUser())
  • show a chart to the user (see showChartToUser())
  • show a Java Swing component to the user (see showComponentToUser())
  • show a list of predefined items containing references to Gedcom objects to the user (see showItemsToUser())

And they can ask for input:

  • ask the user for a directory (see getDirectoryFromUser())
  • ask the user to select an entity (e.g. Individuals, see getEntityFromUser())
  • ask the user to make a choice (e.g. yes/no, see getOptionFromUser())
  • ask the user to enter a value (e.g. name as text, see getValueFromUser())

All of these methods are defined in the super-class Report that every custom Report needs to extend.

Compiling Reports

Since Reports are written in Java they have to be (manually) compiled before they can be used through GenJ. Run one of the compile scripts in the directory containing all reports for that.
